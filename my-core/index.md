# Index — tools, key files, guides

Read this when you need to find a tool or guide. One line each.

## Tools
- bash — run shell commands (full root access)
- read_file — read a file
- write_file — write content to a file
- read_config — safe config values (human_email, etc.); never exposes API keys
- verify_security_answer — check a sender's answer to your security question
- done_for_now — end this loop with a short summary
- search_files — search journals, post-its, guides by keyword
- switch_model — switch provider/model for the next N loops
- check_providers — list available providers and current model
- clear_model_preference — clear any switch_model preference
- workshop_run — run a command in Firejail sandbox (my-workshop/)
- web_fetch — fetch a web page (Scrapling); for JS/login use bash + Playwright
- launch_application — open URL, file, or app by name (allowlist: my-core/launch-allowlist.txt)

## Key files
- my-core/my-wake-state.md — briefing for the next loop; keep under 80 lines / 4000 chars
- my-core/my-post-its.md — short timestamped notes; rotate when large (see post-its-and-rotation.md)
- my-core/current-task.md — plan and progress for multi-loop tasks
- my-core/my-soul.md — your essence (in system prompt)
- my-core/my-personality.md — your voice (in system prompt)
- my-core/launch-allowlist.txt — allowed URLs/apps for launch_application
- my-core/theloop.md — your rhythm (five steps, one rule)
- my-core/bootstrap.md — first-boot steps
